> Model Design

사용 데이터

1. TUNiB의 DKTC 데이터
(Dataset of Korean Threatening Conversations)

데이터 출처: https://github.com/tunib-ai/DKTC

2. AI-Hub의 주제별 텍스트 일상 대화 데이터

데이터 출처: https://www.aihub.or.kr/

LSTM을 활용한 딥러닝 모델

직장 내 일상 대화 / 괴롭힘 대화로 이진 분류

0(괴롭힘 대화)~1(일상 대화) 사이의 확률 반환

단, 모델의 민감도를 고려하여 확률 0.1 이하만을 괴롭힘으로 판별

> Data reordering

데이터를 발화자 기준으로 재정렬

기존 데이터셋은 모든 발화가 개행문자(\n)로 연결되어 있음

ex. 발화1 \n 발화2 \n 발화3 ...

이를 질문-대답 형식으로 바꾸어 활용하였음

ex. 발화1-발화2, 발화2-발화3, 발화3-발화4 ...

> Data Parsing /
Preprocessing

데이터 정제

Konlpy 라이브러리의 Kkma를 이용한 데이터 파싱,
특정 품사를 지정하여 활용하였음

일반명사, 고유명사, 대명사, 동사, 형용사,
긍/부정 지정사, 감탄사, 일반 부사

> Data Vectorization

자연어 데이터를 숫자로 변환, 벡터화

학습/테스트용으로 데이터를 분리하였음

2회 이상 등장한 단어를 기준으로 데이터 정제

자연어를 숫자로 변환 후,
문장 길이 최빈값을 기준으로 패딩 처리
